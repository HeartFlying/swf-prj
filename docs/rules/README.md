# 项目规则文档

本目录包含项目开发规则和最佳实践，用于指导开发工作并避免重复犯错。

## 文档列表

| 文档 | 说明 | 用途 |
|------|------|------|
| [testing-anti-patterns.md](./testing-anti-patterns.md) | 测试反模式 | **必须避免**的问题清单 |
| [testing-best-practices.md](./testing-best-practices.md) | 测试最佳实践 | **建议遵循**的操作指南 |

## 使用方式

### 开发前
1. 阅读 `testing-best-practices.md` 了解正确的测试方法
2. 查看 `testing-anti-patterns.md` 了解需要避免的问题

### 代码审查时
1. 对照反模式检查清单，确保没有犯已知错误
2. 确认测试覆盖了关键路径

### 遇到问题时
1. 查看反模式文档，确认是否是已知问题
2. 如果是新问题，添加到反模式文档

## 规则更新

当发现新问题时：
1. 在 `testing-anti-patterns.md` 中添加新的反模式
2. 在 `testing-best-practices.md` 中添加对应的正确做法
3. 更新本 README 的相关说明

## 关键原则

- **反模式** = 项目中实际发生过的问题，必须避免
- **最佳实践** = 建议的操作方式，用于对比参考
- **两者对照阅读** = 理解"为什么"和"怎么做"
